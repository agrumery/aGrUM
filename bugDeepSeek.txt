MED-14: No endianness handling in GumBinaryIO Lines: 73, 87 Bug: uint64_t is read/written as raw bytes in native endianness. Files from big-endian platforms are misinterpreted on little-endian and vice versa.

MED-17: const_cast on hash table keys allows invariant-breaking modification hashTable.h line: 295 Bug: HashTableBucket::key() uses const_cast to return non-const reference. Modifying a key changes its hash, breaking hash table invariants.

MED-19: Listener copy constructor not exception-safe (BasicSignaler) signaler.h lines: 181-188 Bug: If clone() or push_back() throws partway through, already-cloned connectors are left attached but the destructor won't run (constructor threw), leaking dangling pointers.

MED-20: FixedAllocator heap overflow on blockSize * numBlocks wraparound fixedAllocator_inl.h line: 68 Bug: new unsigned char[blockSize * numBlocks] can allocate a smaller buffer than expected if the multiplication overflows.

MED-22: Formula division by zero and log/sqrt of negative not checked formula_inl.h lines: 194, 218-228 Bug: Division operator, log, log2, sqrt all accept potentially invalid inputs without validation. NaN propagates silently.

MED-26: MeekRules edge erased up to 3 times; i_j && j_i handles inconsistently MeekRules.cpp lines: 214, 218-219, 223-224 Bug: NOTE: eraseEdge is a no-op for non-existent edges; no crash but duplicate calls removed (fixed). Remaining concern: when i_j && j_i both true, both arcs added (double-headed arc), handled downstream by _orientDoubleHeadedArcs_.

MED-27: MeekRules tail_head && head_tail case falls through unhandled MeekRules.cpp lines: 134-155 Bug: When there is a directed path in both directions, none of the if-branches execute. Handled by nuclear option (erase both arcs). Design limitation, not crash.

MED-28: DBCell operator== compares union members across types (NaN edge case) DBCell_inl.h lines: 159-164 Bug: For REAL type, if val_real != from.val_real, it falls through to val_integer == from.val_integer. For NaN, IEEE 754 says NaN != NaN, but the integer bit-pattern comparison would say NaN == NaN if they have the same bit pattern.

MED-30: DBRowGeneratorEM potential OOB if columns_of_interest has duplicates lines: 276-302 Bug: If columns_of_interest_ contains duplicates, i could be incremented past missing_cols.size() causing OOB access.

MED-31: DimFunctionGraph copy constructor does not copy internal state (various files) Bug: MultiDimFunctionGraph copy construction may not properly duplicate the function graph tree structure.

MED-36: intPow silent integer overflow pow_inl.h lines: 57-66 Bug: unsigned long out *= base can overflow silently.
